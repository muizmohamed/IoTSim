# Security Policy

## Supported Versions
We release patches for security vulnerabilities in the following versions:
- Latest main branch
- Latest stable release

## Reporting a Vulnerability
If you discover a security vulnerability, please report it responsibly:
- Email: muizmohammed123@gmail.com
- Do not open a public GitHub issue for security concerns.
- Provide detailed information to help reproduce and fix the issue.

## Security Best Practices
- Keep dependencies updated.
- Run tests before submitting pull requests.
- Do not commit sensitive information (passwords, tokens, private keys).
- Follow secure coding guidelines (input validation, error handling).

## Responsible Disclosure
We appreciate responsible disclosure of vulnerabilities. We will acknowledge your report, investigate, and provide updates on remediation.
